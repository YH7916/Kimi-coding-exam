2. **补 CI**
   加 GitHub Actions：`ruff`、`mypy`、`pytest`、`python scripts/evaluate.py`。
   这对简历很有用，因为你可以说“有自动化质量门禁和离线评测”。

3. **README 改成作品集风格**
   现在 README 更像题目说明。建议顶部加：
   - 架构图
   - 一键启动
   - API 示例
   - v1/v2/v3 能力对比
   - eval 结果表
   - 真实 LLM provider 配置说明
   - 截图或 GIF

4. **补一张 Agent 架构图**
   面试时非常好讲：

   `HTML SOP -> Parser -> Chunking -> BM25 + Vector Retrieval -> RRF Fusion -> Agent Candidate Context -> readFile Tool -> Evidence Extraction -> Final Answer + Trace`

   这会把项目从“我写了个问答系统”升级成“我理解 Agent/RAG 工程链路”。

5. **增强 eval，不只是 README case**
   现在 13 个 case 全 1.00，很漂亮，但数量偏少。可以加：
   - hard negative：问不存在的流程时不要乱答

   - ambiguous query：同时涉及 DB/SRE/Security 的问题
   - real-provider smoke 独立保留，不默认跑

6. **加 Docker 或最小部署说明**
   不一定要复杂部署，但一个 `Dockerfile` 或 `docker compose up` 能大幅提高“可交付”观感。

7. **面试故事再包装**
   简历上不只写“用了 RAG/Agent”，而是突出你解决了什么工程问题：
   - 如何让 Agent 不乱读文件
   - 如何让检索结果可解释
   - 如何在没有 API key 时仍可离线验收
   - 如何用 fallback 保证 demo 稳定
   - 如何用 eval 防止提示词/检索改动退化
