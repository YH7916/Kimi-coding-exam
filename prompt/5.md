**我会先拆前端，不急着动后端**

后端现在已经够清楚，真正还显胖的是 `frontend/app.js`。下一提交最好只做一件事：把 `app.js` 拆成 ES modules，然后 `index.html` 里改成：

```html
<script type="module" src="/static/app/main.js"></script>
```

这样不引入 Vite/React/打包工具，交付风险最低，也保留“打开 FastAPI 就能跑”的简单性。

**拆分顺序**

1. 先抽纯函数：`markdown.js`、`sse.js`、`config.js`。
2. 再抽状态和存储：`state.js`、`history.js`、`settings.js`。
3. 再抽 API：`api.js`。
4. 最后抽 render：`render/chat.js`、`render/search.js`、`render/shell.js`。
5. `main.js` 只负责组装和事件绑定。

**边界原则**

- `api.js` 不碰 DOM。
- `render/*` 尽量只返回 HTML 字符串或更新局部 DOM。
- `state.js/history.js` 不发请求。
- `markdown.js` 完全纯函数。
- `main.js` 可以“脏一点”，负责 glue code，但不要堆业务细节。

这样拆完之后，代码会像一个真实小型 Web app，而不是一个竞赛题临时脚本。
