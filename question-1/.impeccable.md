## Design Context

### Users
This is an interview/demo On-Call Copilot for reviewers and engineers who need to compare v1 keyword search, v2 RAG retrieval, and v3 tool-using Agent behavior against a small SOP corpus.

### Brand Personality
Calm, precise, restrained. The UI should make the backend capability feel credible without pretending the dataset is larger than it is.

### Aesthetic Direction
Minimal light interface inspired by Vercel, Apple, and the ChatGPT home screen: centered work surface, quiet navigation, generous whitespace, and compact result presentation. Avoid dashboard density, repeated explainer cards, marketing hero composition, and decorative gradients.

### Design Principles
- Keep the first screen focused on one action: ask or search.
- Reuse one interaction model across v1, v2, and v3, with only capability-specific result details.
- Prefer subtle typography, spacing, and borders over decorative UI.
- Show v3 tool calls and evidence clearly, but keep v1/v2 restrained.
- Match the small data volume: fewer panels, fewer prompts, less explanatory copy.
